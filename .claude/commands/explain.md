# Explain Code Command

Provide a clear explanation of how code works.

## Steps

1. Read the specified file or code block
2. Identify the overall purpose
3. Break down into logical sections
4. Explain each section in plain language

## Explanation Structure

```markdown
## Overview

[What this code does in 1-2 sentences]

## Purpose

[Why this code exists, what problem it solves]

## How It Works

### Step 1: [Section Name]
[Explanation of what this section does]

```code
[relevant code snippet]
```

### Step 2: [Section Name]
[Explanation continues...]

## Key Concepts

- **[Concept 1]**: [Brief explanation]
- **[Concept 2]**: [Brief explanation]

## Data Flow

[Describe how data moves through the code]

Input → [Process 1] → [Process 2] → Output

## Dependencies

- `[dependency]`: Used for [purpose]

## Potential Issues / Gotchas

- [Thing to be aware of]

## Example Usage

```code
[How to use this code]
```
```

## Levels of Detail

Based on user's expertise level:

### Beginner
- Explain basic concepts
- Define technical terms
- Use analogies
- More examples

### Intermediate
- Focus on logic and patterns
- Explain design decisions
- Compare alternatives

### Expert
- Focus on edge cases
- Performance implications
- Security considerations
- Architectural context

Default to intermediate unless specified.

## Rules

- Use plain language, avoid jargon
- Include code snippets for context
- Use visual aids (ASCII diagrams) when helpful
- Be concise but thorough
- Highlight any "magic" or non-obvious behavior
